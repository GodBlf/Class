# 学习心得
- 完全基于http协议底层原理,每次来回是因为js代码在不停执行
- session也是基于https 原理就是把每次request response返回的所有cookie之类的保存起来
- js脚本如何发送数据,服务端返回到本地http接口,js通过组装这些接口实现发送数据,爬到网站接口js文件来请求服务器
- 返回的cookie是识别身份用的
- 二维码登录是pc端在不断轮询或者建立了websocket双向通讯
