在 Windows 的“Win + R”（运行）框中自定义命令，主要有以下几种方法，可以让你输入自定义的关键词快速启动程序、脚本或打开路径：

---

## 1. 修改系统环境变量（添加到 PATH）

把你想执行的程序或脚本所在目录添加到系统环境变量 PATH 中，然后在“Win + R”中直接输入程序名称即可运行。

### 步骤：

1. 右键“此电脑” → 属性 → 高级系统设置 → 环境变量
2. 找到系统变量里的 **Path** ，编辑，将自定义程序所在文件夹路径添加进去（比如 `C:\MyTools`）。
3. 确认保存并重启资源管理器或重新登录。
4. 将你的.exe、.bat、.cmd脚本放到该文件夹，用文件名作为命令即可执行。

---

## 2. 创建快捷方式放入系统文件夹并重命名

Windows 在“Win + R”时，会自动搜索系统路径里的一些特殊文件夹资源。

### 可以放入：

- `%windir%\System32`
- `%windir%\SysWOW64`
- 或放入你自己新建的文件夹并加入 PATH

如果你的程序或快捷方式名称在这些路径中唯一，那么Win+R直接输入名称即可启动。

---

## 3. 使用注册表添加自定义命令 (HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\App Paths)

注册表中 `App Paths` 允许你为一些程序定义简写命令，Win+R 运行时会先在这里查找。

### 用法：

1. 按 Win+R 输入 `regedit` 打开注册表编辑器。
2. 跳转到路径：

   ```
   HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\App Paths
   ```
 
3. 右键 `App Paths` → 新建 → 键，比如命名为 `mytool.exe` （命令包含后缀.exe，方便识别）
4. 在新建键的默认值中，写入程序的完整路径，比如：
   ```
   C:\MyTools\mytool.exe
   ```
5. 可选：新建字符串值 `Path`，设置程序所在的路径
6. 关闭注册表后，即可直接在“Win + R”中输入 `mytool` （省略.exe）运行该程序。

---

## 4. 利用批处理文件（.bat）+ 环境变量

将常用命令写成 `.bat` 文件，放到某个自定义目录中（比如 `C:\MyCmds`），再将这个目录添加到 `PATH` 环境变量。

然后“Win + R”输入你批处理文件名即可执行自定义的一系列操作。

---

## 5. 使用第三方工具

比如 [Clink](https://mridgers.github.io/clink/) 或 [Cmder](https://cmder.net/) 等增强命令行工具，可以增强运行窗口的功能，对复杂命令有更好支持。

---

## 总结

| 方法                  | 实现形式                       | 适用场景                       |
|-----------------------|------------------------------|------------------------------|
| 增加 PATH              | 程序/脚本放目录 / 环境变量    | 简单命令或程序快速启动         |
| 注册表 App Paths      | 注册表直接绑定程序路径         | 简单便捷，命名灵活             |
| 快捷方式放特定目录     | 创建快捷方式                   | 非可执行文件静态链接           |
| 批处理文件自定义       | 脚本批处理                    | 运行一系列自定义操作           |
